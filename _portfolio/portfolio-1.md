excerpt: "An intelligent, multi‑agent RAG (Retrieval‑Augmented Generation) system that extends the CIQUAL food search engine with adaptive reasoning, query refinement, and autonomous search strategies.<br/><br/>Unlike the traditional fixed‑pipeline RAG, this system employs a collaborative agent architecture: an Orchestrator Agent coordinates the workflow, a Query Refiner Agent detects typos and rewrites ambiguous queries, a Search Agent executes and evaluates multiple retrieval attempts, and a Response Agent generates final answers with citations and follow‑up suggestions. The agent loop can retry with improved queries when initial searches return weak results, making the system robust to misspellings and poorly phrased questions.<br/><br/>Built with Python, PostgreSQL + pgvector, SentenceTransformers, Cross‑Encoders for reranking, SymSpell for spelling correction, and Ollama for local LLM inference. The modular design includes conversation memory for multi‑turn interactions.<br/>As demonstrated by the query <b><q>What is the protien content of the mixed salad with fihs?</q></b>, which intentionally contains the misspellings <b><i>protien</i></b> (instead of protein) and <b><i>fihs</i></b> (instead of fish), the agent can automatically detect and correct spelling errors. It can also reformulate or suggest improved queries to optimize retrieval and enhance the quality of the LLM&quot;s responses.
